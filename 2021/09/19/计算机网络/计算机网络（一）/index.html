计算机网络（一）

计算机四级考试 –计算机网络

注：此文章仅针对计算机四级考试中计算机网络的主要考点，适合对计算机网络有一定了解并准备考试的兄弟们阅读，也可作为知识点的完善或扩充，如有不足，恳请指正
注：本文部分图片来自未来教育计算机网络，谨此感谢
基本要求：

  • 理解计算机网络的基本概念
  • 掌握局域网的基本工作原理
  • 掌握TCP/IP及其相关协议
  • 掌握较为新型的网络技术应用
  • 理解网络管理与网络安全原理

第一章 网络技术基础

知识点：

  • 计算机网络的形成与发展
  • 计算机网络的基本概念
  • 数据包交换与虚拟电路交换
  • 网络体系结构与网络协议

    一.计算机网络的形成与发展

  1. 计算机网络的发展阶段

    • 第一阶段
      • 数据通信技术成熟
      • 分组交换概念提出
    • 第二阶段
      • ARPANET的成功运行
      • TCP/IP协议的广泛应用
    • 第三阶段
      • OSI参考模型的研究
      • TCP/IP协议推动力Internet的发展
    • 第四阶段
      • Internet应用、无线网络与网络安全技术研究的发展

  2. ARPANET（阿帕网络）（广域网）

    • 美国国防高级研究计划局（DARPA） 要求新的网络在遭遇核战争或自然灾害时，如果部分网络设备或通信线路遭到破坏，网络系统仍能利用剩余的网络设备与通信线路继续工作—-称为可生存系统。
    • ARPANET采取分组交换的思想，分为两个部分：通信子网和资源子网，通信子网的转发节点用小型机实现，称为接口报文处理器（IMP）。

    APARNET发展

    主要贡献

    • 开展对计算机网络定义与分类方法的研究
    • 提出资源子网与通信子网的二级结构概念
    • 研究分组交换协议与实现技术
    • 研究层次型网络体系结构的模型与协议体系
    • 开展TCP/IP协议与网络互连技术的研究

  3. 宽带城域网与三网融合技术的发展

    • 宽带城域网结构：核心交换层、汇聚层、接入层
    • 三网融合：计算机网络、电信网、电视网

  4. 物联网技术的形成与发展
    物联网是在Internet技术基础上，利用射频标签、无线传感、光学传感等感知技术，自动获取物理世界的各种信息，通过三网融合形成的新一代网络系统（IPv6），构建覆盖人与人、人与物、物与物的智能信息系统。
    物联网实现了物理世界与信息世界的无缝连接。
    物联网与Internet关系：物联网应用系统接入到当前的Internet。

  5. 网络操作系统的发展
    Windows操作系统：

    • Microsoft公司推出，带有网络功能，早期推出的操作系统：Windows3.1，它在DOS环境中增加了GUI图形用户界面。后来又推出Windows for Workgroup对等式操作系统。
    • Windows NT + Windows NT 3.1 ：采用NT内核的32位的操作系统，提供全面网络服务。 + Windows NT 3.5 ：性能、安全、网络管理有所提高，降低对微型机配置要求。 + Windows NT 4.0 ：采用客户机/服务器（C/S）的工作模式。
    • Windows 2000 ：提供文件打印、web等服务，功能强大、易配置、安全性好。
    • Windows XP ：安全性、可靠性、管理功能、即插即用、界面简单。
    • Windows 2003
    • 2009年Windows7：支持触控技术的桌面操作系统。
    • 2012年Windows8：采用Modem模式的用户界面，屏幕触控。
    • 2015年Windows10

    Unix操作系统

    • 1969年，Unix第一个版本V1，汇编语言编写，目的是为开发新软件的程序员提供工具(闭源)，各个公司的Unix操作系统包括：IBM公司的AIX系统、Sun公司的Solaris系统、HP公司的HP-UX系统。

    Linux操作系统

    • 常见发行版：RedHat、Ubuntu、蓝点、红旗Linux…
    • 完全免费，适合作为Internet服务平台，价格低、开源、安装配置简单。

二.计算机网络的基本概念

1. 计算机网络：以能够相互资源共享的方式互联起来的自治计算机系统的集合

  • 特征1：实现计算机资源的共享
  • 特征2：互联的计算机是多台分布在不同地理位置的独立的“自治计算机”，之间没有明确的主从关系，每台计算机可联网或脱网工作
  • 特征3：联网之间的计算机必须遵循共同的协议

2. 不同网络术语的区别与联系

  • computer network：计算机网络，有广域网、城域网、局域网或个人区域网等
  • internet：表示网络互联，将多个计算机网络互联成大型网络系统
  • Internet：因特网、互联网，是由很多种网络互联起来的网际网
  • Intranet：内部的专用网络系统

3. 计算机网络分类：

（1）广域网技术

  • 广域网（WAN）又称远程网，地理范围几十千米到几千千米
  • 覆盖一个国家、地区或横跨几个州，国际性
  • 将分布在不同地区的计算机系统、城域网、局域网互联，实现资源共享
  • 公共数据网络，主要特征是宽带核心交换技术，传输介质：光纤

（2）城域网技术
光纤传输介质，45150Mbps高传输速率，支持数据、语音与视频综合业务的数据传输，可覆盖50100km

  • 光纤传输，传统电信、有线电视与IP业务融合成了宽带城域网的核心业务
  • 核心设备：高端路由器与多层交换机
  • 关键：扩大宽带接人的规模与服务质量
    + 管理平台：用户认证与接入管理、业务管理、网络安全、计费能力、IP地址分配、QoS保证等 + 业务平台：为用户提供Internet接入业务、数据专线业务、话音业务、视频与多媒体业务、内容提供业务等 + 网络平台结构：核心交换层、边缘汇聚层、用户接入层

（3） 局域网技术
有限范围内的计算机、终端于外部设备互联成网
主要有两种：共享局域网与交换局域网
特征：

  • 覆盖范围有限，高传输速率、低误码率，一般属于一个单位所有，易建立、维护与扩展
  • 决定性能的三个因素：拓扑、传输介质、介质访问控制方法

（4）个人区域网技术

  • 无线个人区域网络（WPAN），无线通信技术，覆盖距离：10m以内
  • 无线传感器网使用的无线通信技术主要包括IEEE802.11标准的WLAN、IEEE802.15.4标准的无线个人区域网（6LoWPAN）、蓝牙技术（Bluetooth）（10m内通信，低功耗）、ZigBee技术（面向自动控制、数据采集与控制节点多等等）

4. 计算机网络拓扑结构

  • 星形拓扑：节点通过点-点通信线路与中心节点连接，中心节点控制全网通信，任何两点间通信需要中心节点，结构简单，易于实现与管理，但中心节点是全网性能与可靠性的瓶颈，中心节点故障会导致全网瘫痪
  • 环形拓扑：闭合回路，任何节点都存在隐患
  • 总线型拓扑：广播式发送与接收数据，结构简单，但存在冲突即介质访问冲突问题
  • 树形拓扑
  • 网状拓扑：任意，无规律，系统可靠性高，结构复杂，必须采用路由选择算法、流量控制与拥塞控制方法

5. 计算机网络传输特性参数

  • 数据传输速率：每秒钟传输构成数据的二进制比特数（bps）
  • 误码率：二进制码元在数据传输系统中被传错的概率，P=Ne/N，是正常工作状态下的传输可靠性参数，在数据传输速率确定后，误码率越低，传输系统设备越复杂，造价越高

三.数据报交换与虚电路交换

1.数据报方式(主要方式)

数据报：一种报文分组存储转发方式，分组传输前不需要在源主机与目的主机预先建立“线路连接”，每个分组可独立选择路径传输

特征：可通过不同路径，会出现乱序、重复等现象，每个分组必须携带目的地址和源地址,延迟较大，不适合长报文、会话通信

2.虚电路方式

在发送分组前，发送方与接收方建立一条逻辑连接的虚电路

特征：源与目的主机间预习建立虚电路,虚电路顺序传输，无乱序等现象，分组经过路由器只需差错检验，不需路由选择,路由器可与多个主机之间通信建立多条虚电路

四.网络体系结构与网络协议

网络体系结构

网络协议概念：语法（用户数据与控制信息的结构和格式），语义（控制信息意义），时序（事件发生顺序）
网络体系结构：层次结构优点（各层相互独立，灵活性好）

一.OSI参考模型

开放系统互联（open system Internetwork，OSI），由国际标准化组织（ISO）确定，网络互联的七层框架，详细规定每层功能，实现开放系统环境中的互联性，互操作性与应用的可移植性

网络主机具有相同层次；不同主机同层次功能相同；同一主机相邻层通过接口通信；上层可使用下层提供的服务，并向上层提供服务；不同主机同等层次通过协议进行同层通信

  • 物理层：最底层，实现比特流的透明传输数据传输单位为比特
  • 数据链路层：建立数据链路，采用差错控制与流量控制方法，传输单位是帧
  • 网络层：通过路由选择算法为分组选择合适传输路径，实现流量控制、拥塞控制与网络互联，传输单元是分组（packet）
  • 传输层：提供可靠端到端（end to end）连接与数据传输服务，传输单元是报文（message）
  • 会话层：会话主机连接与通信
  • 表示层：通信系统之间的数据格式变换、数据加密解密、压缩恢复等
  • 应用层：应用程序之间的通信控制
二.TCP/IP参考模型与协议

TCP/IP协议规定了计算机通信所使用的协议数据单元、格式、报头与相应动作
特点：开发协议标准，独立于计算机硬件与操作系统，独立于网络硬件，适用Internet，统一网络地址分配，标准化应用层协议
TCP/IP与OSI

  • TCP/IP:
    • 主机-网络层：使用网络协议通过网络发送和接受IP分组，提供接口
    • 互联网络层：处理来自传输层的数据发送请求；处理接受分组；处理网络的路由选择、流量控制、拥塞控制
    • 传输层：负责会话进程之间建立和维护端到端连接，传输层协议：传输控制协议（TCP）和用户数据报协议（UDP）
    • 应用层：远程登录协议（Telnet),文件传输协议（FTP），简单邮件传输协议（SMTP），超文本传输协议（HTTP），域名服务（DNS），简单网络管理协议（SNMP），动态主机配置协议（DHCP）
